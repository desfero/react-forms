What is a controlled component?

  • Controlled components have functions to govern the data going into them on every onChange event, rather than grabbing the data only once, e.g. when a user clicks a submit button.
  • This 'governed' data is then saved to state (the parent/container component's state).
  • Data displayed by a controlled component is received through props passed down from it's parent/container component.

Advantages:

  • You are set up to easily validate user input.
  • You can dynamically render other components
    based on the value of the controlled component.
    For example, the value a user selects from a dropdown (e.g. 'dog' or 'cat') can control which other form components (e.g. a checkbox set of breeds) are rendered in the form.


Disadvantages:

  • The amount of code you have to write.
    The amount of code you have to write: You need a state property to pass to the form element as props, and you need a function to update the value of this property.
  • Possibility of CSS keylogger attack.

What is a uncontrolled component?

  • Form data are handled by the DOM.
  • 'ref' is used to get and set form values.

So, the difference?

Controlled component can respond to input changes immediately; for example, by:

  • in-place feedback, like validations
  • disabling the button unless all fields have valid data
  • enforcing a specific input format, like credit card numbers
